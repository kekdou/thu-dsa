## 算法思路和实现要点

首先是关于树的存储，采用正向表法的方式，利用 idx 数组记录每个节点子节点在 children 数组中的起始位置

子树内力值之和，先找出根节点，然后从根节点开始利用 DFS 对每个节点进行遍历并计算子树和  

搜索引路者，对于正统弟子 u（师父为 p），如果 val[p] < val[u]，则直接跳向 master[p]，直到找到比 u 大的节点或路径终点

用以上逻辑对每个节点进行 DFS，正统弟子继承路径，非正统则设为空

## 渐进时间复杂度的分析，包含过程

- 计算子树和 FillSum: 采用 DFS，每个节点和边仅访问一次，因此复杂度为 O(n)
- 计算引路者 Solve: 
  - 内部 while：由于每次查找会跳跃至上一个节点 p 的 master，因此如果节点 A 被节点 B 跳过，意味着 val[A] < val[B]。后续节点 C（B 的弟子）在查找时，会直接从 B 开始跳跃，根本不会再访问到 A，所以 while 循环中执行的总次数，不会超过 N。
  - 外部 DFS，时间复杂度为 O(n)
  - 总时间复杂度为 O(n)

综上所述，时间复杂度为 O(3n) = O(n)

## 渐进空间复杂度的分析，包含过程

利用正向表储存树，辅助索引数组 idx 需要 O(n)  
计算子树和 sum 需要 O(n)  
计算根节点的辅助数组 not_root 需要 O(n)  
DFS 搜索需要的递归栈空间，最坏情况下退化为一条链，需要 O(n)  

因此总体空间复杂度为 O(n)

## 遇到的困难和解决方法

关于树的存储想了很长时间，因为不能使用 vector，如果强行采用 N * N 的二维数组会空间超限，后面记起图论中正向表的内容，能很好解决这个问题

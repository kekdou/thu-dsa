### 01.cpp

1. 错误类型：**Runtime Error**  
2. 错误原因：
   - 代码调用 `play(left - 1)` 时没有对传入的 rank 进行边界检查，导致传入的可能为负值

```C++
AA
1

```

### 03.cpp

1. 错误类型：**Time Limit Exceeded**
2. 错误原因：
   - string 的 insert 和 erase 方法都是 $O(n)$ 操作
   - 如果每次都在最左侧操作的话需要 $O(m * n)$，对于大规模数据无法通过
3. 发现方法：观察法 + 排除法
4. 如何触发：将数据规模拉到最大的 n = m = 500000，并且让每一次 insert 和 erase 都在最左侧执行，程序就会运行超时  

使用如下代码生成测例

```C++
int main() {
    ofstream fout("code/03.in");
    for (int i = 0; i < 125000; i++) {
        fout << "AABB";
    }
    fout << "\n";
    fout << "500000" << "\n";
    for (int i = 0; i < 250000; i++) {
        fout << "0 A" << "\n";
        fout << "0 B" << "\n";
    }
    fout.close();
    return 0;
}
```

### 06.cpp

1. 错误类型：**Wrong Answer**
2. 错误原因：
   - 代码 74 行在使用 `memmove(cur + 1, cur, succ_len)` 时没有考虑到当前块元素数量大于 plen_bound 的情况
   - 并且整个代码都没有考虑 plen_bound
   - 就以上问题而言，如果一个块一直插入，并且下一个块有元素的话，超出最大容量后上一个块最右侧的元素会覆盖掉下一个块左侧的元素
   - 在 p2a 还原序列时也会出现问题，从而引发整个序列错误
3. 发现方法：观察法 + 调试
4. 如何触发：初始序列被分为两个块，不断向第一个块插入而不发生消除，导致第二个块的元素被覆盖，引发错误

使用如下代码生成测例

```C++
int main() {
    ofstream fout("code/06.in");
    for (int i = 0; i < plen_target; i++) {
        fout << "AB";
    }
    fout << "\n";
    fout << plen_target + 1 << "\n";
    for (int i = 0; i < plen_target + 1; i++) {
        char c = (i % 6) + 'C';
        fout << "0 " << c << "\n"; 
    }
    fout.close();
    return 0;
}
```

### 07.cpp

1. 错误类型：**Wrong Answer**
2. 错误原因：
   - 代码在 96 行使用 `if` 来退回上一个块，但没有考虑到空行的情况，应该使用 `while`
   - 如果存在空行，将会导致后续删除出现异常
3. 发现方法：观察法 + 调试 + 对比法 (对比 `08.cpp`)
4. 如何触发：先生成三块的数据，然后将第二块消除成空块，在对第一块最右侧和第三块左侧进行消除，由于在检测边界时没有跳过第二块空块，导致消除异常出错

使用如下代码生成测例

```C++
int main() {
    ofstream fout("code/07.in");
    for (int i = 0; i < plen_target; i++) {
        char c = (i % 2) + 'A';
        fout << c;
    }
    for (int i = 0; i < (plen_target >> 1); i++) {
        char c = (i % 2) + 'C';
        fout << c << c;
    }
    for (int i = 0; i < plen_target; i++) {
        char c = (i % 2) + 'B';
        fout << c;
    }
    fout << "\n";
    fout << plen_target / 2 + 1 << "\n";
    for (int i = 0; i < (plen_target >> 1); i++) {
        char c = (i % 2) + 'C';
        fout << plen_target << " " << c << "\n";
    }
    fout << plen_target << " " << 'B' << "\n";
    return 0;
}
```

**角色设定：**
你是一位精通算法竞赛（ACM/ICPC）和 C++/数据结构的专家。

**任务目标：**
请帮我检查并修正以下这段模拟“祖玛（Zuma）”游戏逻辑的代码。代码的目标是根据输入的操作序列，实时维护珠子轨道并处理连锁消除逻辑。

**逻辑规则：**
1. **插入：** 在给定的位置 $k$ 插入一颗颜色为 $\Sigma$ 的珠子。
2. **连锁消除：** 插入后，如果该位置及其相邻区域出现**连续 3 个或以上**同色的珠子，则这些珠子必须被消除。
3. **连锁反应：** 消除后，左右两端的珠子会重新接头。如果接头处再次形成**连续 3 个或以上**同色的珠子，则继续消除，直到不再满足消除条件为止。

**输入/输出规范：**
- **输入：** 初始字符串、操作总数 $m$、以及 $m$ 组插入位置 $k$ 和颜色 $\Sigma$。
- **输出：** 最终的珠子序列。若为空则输出换行。
- **注意：** 字母、数字混合输入，且初始序列可能为空，需要使用足够鲁棒的读入方式

**性能挑战（重点）：**
- 数据规模：$n, m \le 500,000$。
- 时间限制：1.5 秒。
- 空间限制：256 MB。
- **注意：** 使用传统的 `std::string::insert` 或 `std::vector::erase` 复杂度为 $O(N)$，在 $5 \times 10^5$ 规模下会导致 $O(N \cdot M)$ 的总复杂度，必然超时。

**请根据以上要求，找出上传的代码的最严重的错误：**

错误类型的严重程度按如下排列：  
Wrong Answer > Runtime Error > Time Limit Exceeded = Memory Limit Exceeded

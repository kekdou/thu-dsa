## 算法思路和实现要点

使用 KD-Tree 求取欧氏距离最近点    
1. 读入 n 个 d 维点，递归建树：第 `depth` 层按 `axis = depth % d` 选划分维度，将该维排序后取中位点作为当前结点，左右子区间递归构建。  
2. 查询时执行分支限界：先访问离目标点可能更近的子树，再判断是否需要访问另一侧子树，判断条件为
 `(target[axis]-node[axis])^2 < 当前最优距离`，若分隔面到目标点的最小距离都不能改进答案，则剪枝  
3. 每访问一个结点都判断更新一次最优解  

该方法在低维数据上通常远快于朴素 O(n) 扫描。

## 渐进时间复杂度的分析，包含过程

设点数为 n、维度为 d、查询数为 q   
- `build()`: 建树递归式为 $T(n) = 2T(n/2) + O(nlogn)$，因为每层都对当前子数组重新排序（代码使用 `sort`，不是线性选中位数）。按主定理得 $T(n)=O(n\log^{2} n)$ 
- `query()`: 单次查询中，每访问一个结点要算一次 d 维距离，代价 O(d)，若树较平衡且剪枝有效，访问结点期望为 O(log n)，故平均时间复杂度为 O(dlogn)，最坏情况下会访问全部结点，为 O(dn) 

程序总时间复杂度为：$O(n\log^{2} n + qd\log n)$（平均）到 $O(n\log^{2} n + qdn)$（最坏）。

## 渐进空间复杂度的分析，包含过程

1. KD-Tree 结点共 n 个，每结点含左右指针与轴信息，额外结构空间 O(n)
2. 建树时每层会构造左右子向量 `lc/rc`，沿单条递归路径，同时存在的子向量总元素量为  
$n + n/2 + n/4 + ... < 2n$，故最大辅助空间为 O(n)，递归栈深度平衡时 O(log n)  
3. 查询阶段仅使用递归栈与常数状态 `Res`，栈深度为树高，平衡时 O(log n)、最坏退化 O(n)

因此总空间复杂度为 O(n)。
